# Setup für den Kurs

## 1. Dateizugang mit GitHub

Damit ihr immer und von überall zugriff auf die Dateien habt, die wir im Tutorium benutzen, liegen sie in ihrem eigenen GitHub Repository.

### Warum Github?
GitHub ist eine Online-Plattform für Entwickler*innen, auf der sie ihren Code hochladen und Versionieren können. Sicherlich gibt es auch andere Plattformen, auf denen man die Dateien zugänglich machen kann, aber GitHub ist der Industrie-Standart in der Entwicklung und Versionierung von Code, deshalb ist es auch inhalt dieses Tutoriums, wie ihr mit GitHub zukünftig zusammenarbeiten könnt.

### Github installieren

## 1. GitHub-Account anlegen

Wenn du noch keinen GitHub-Account hast:

1. Öffne im Browser [https://github.com](https://github.com).
2. Klicke auf `Sign up`.
3. Wähle eine E-Mail-Adresse, einen Benutzernamen und ein Passwort.
4. Bestätige deine E-Mail-Adresse.
5. Melde dich danach bei GitHub an.

Empfehlung:

- Wähle einen Benutzernamen, den du auch im Kurs gut verwenden kannst.
- Notiere dir deinen GitHub-Benutzernamen. Du wirst ihn später brauchen.

## 2. Git installieren

Wichtig: Um im Terminal mit GitHub-Projekten zu arbeiten, brauchst du in der Regel `Git`, nicht nur einen GitHub-Account.

### macOS

1. Öffne das Terminal.
2. Gib diesen Befehl ein:

```bash
git --version
```

3. Wenn eine Versionsnummer erscheint, ist Git bereits installiert.
4. Wenn Git noch nicht installiert ist, bietet macOS meist automatisch die Installation der Command Line Tools an. Bestätige diese Installation.

### Windows

1. Gehe im Browser auf [https://git-scm.com/download/win](https://git-scm.com/download/win).
2. Lade den Installer herunter.
3. Starte den Installer und übernimm für den Anfang die Standard-Einstellungen.
4. Öffne danach `Git Bash` oder das Windows-Terminal.
5. Prüfe die Installation mit:

```bash
git --version
```

### Git zum ersten Mal einrichten

Diese beiden Befehle solltest du einmal auf deinem Computer ausführen. Ersetze die Platzhalter durch deine eigenen Daten.

```bash
git config --global user.name "Dein Name"
git config --global user.email "deine-email@example.com"
```

Prüfen kannst du das so:

```bash
git config --global --list
```
